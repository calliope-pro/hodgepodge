# Article Structure Patterns

Proven article structures for different blog genres on Hodgepodge.

## Universal Structure (All Articles)

All articles follow this basic flow:

1. **Hook** - Grab attention, state the purpose
2. **Context** - Background information
3. **Content** - Main substance of article
4. **Value** - What reader should remember/take away

## Pattern 1: Problem-Solution (Technical)

Best for: Tutorials, troubleshooting, tool guides

### Structure

```markdown
## きっかけ
[What prompted this article? What problem occurred?]

## 状況と原因
[Context: What's the situation? What caused the issue?]

## 解決方法
[Step-by-step solution]
### 手順1
[Detailed first step]
### 手順2
[Detailed second step]

## まとめ
[Key takeaways, what to remember]
```

### When to Use
- Troubleshooting errors
- How-to guides
- Implementation tutorials
- Tool comparisons

### Key Elements
- Clear problem statement
- Reproducible steps
- Code examples with language tags
- Verification that it works

## Pattern 2: Introduction-Body-Conclusion (General)

Best for: Learning notes, summaries, explanations

### Structure

```markdown
## 導入
[What is this about? Why does it matter?]

## {Topic 1}
[First main point]

## {Topic 2}
[Second main point]

## {Topic 3}
[Third main point]

## まとめ
[Main takeaways]
```

### When to Use
- Explaining concepts
- Summarizing learning
- Sharing research
- Overview of topics

### Key Elements
- Clear sections for each topic
- Logical flow from basic to advanced
- Practical examples where applicable

## Pattern 3: Journey-Based (Lifestyle/Diary)

Best for: Personal experiences, reflections

### Structure

```markdown
## きっかけ
[What started this? Background context]

## {Phase 1}
[First part of journey/experience]

## {Phase 2}
[Second part, what changed/happened]

## 振り返り
[What was learned? What would you do differently?]
```

### When to Use
- Daily life observations
- Process documentation
- Personal projects
- Habit building

### Key Elements
- Authentic voice
- Specific details (not vague)
- Honest reflection
- Practical insights for readers

## Pattern 4: Comparison (Choices)

Best for: Tool comparisons, decision guides

### Structure

```markdown
## {Tool A}と{Tool B}の違い
[What are we comparing? Why does it matter?]

## {Tool A}の特徴
[Pros, cons, use cases]

## {Tool B}の特徴
[Pros, cons, use cases]

## どちらを選ぶべきか
[Decision framework based on use case]
```

### When to Use
- Tool comparisons
- Framework choices
- Library evaluations
- Alternative solutions

### Key Elements
- Fair comparison (not biased)
- Specific use cases for each
- Clear recommendation criteria
- Links to official docs

## Pattern 5: Quick Tips (Listicle)

Best for: Multiple quick points, collections

### Structure

```markdown
## {Number}つの{TIPS/HINTS/METHODS}
[What are these tips for?]

### 1. {Tip 1}
[Detailed explanation]

### 2. {Tip 2}
[Detailed explanation]

[Continue for each tip]

## おすすめの使い方
[How to apply these tips]
```

### When to Use
- Quick reference guides
- Best practices
- Tips collections
- Checklists

### Key Elements
- Numbered items
- Scannable format
- Actionable advice
- Application guidance

## Heading Style by Pattern

### Problem-Solution
- `## きっかけ` - What happened
- `## 状況と原因` - Context
- `## 解決方法` - Solution
- `## まとめ` - Summary

### Introduction-Body
- `## 導入` - Intro
- `## {Topic Name}` - Each main section
- `## まとめ` - Conclusion

### Journey-Based
- `## きっかけ` - Beginning
- `## {Phase Name}` - Each phase
- `## 振り返り` - Reflection

### Comparison
- `## {A}と{B}の違い` - Comparison intro
- `## {A}の特徴` - First item
- `## {B}の特徴` - Second item
- `## どちらを選ぶべきか` - Decision guide

## Opening Hooks by Genre

### Technical Posts
- "〇〇を使っていて、✕✕という問題に遭遇しました"
- "新しいツール〇〇を試したところ、驚くほど高速でした"
- "環境構築ではまったエラーと解決方法をまとめました"

### Lifestyle Posts
- "最近、〇〇を始めて気づいたことがあります"
- "日常のちょっとした工夫で、〇〇が改善されました"
- "何気ない行動から、意外な発見がありました"

### Learning Notes
- "〇〇を学んでいて、重要だと感じたポイントを整理しました"
- "ドキュメントを読んでいて分かりにくかった部分をまとめました"
- "初心者がつまずきやすい〇〇の概念を説明します"

## Closing Sections by Genre

### Technical Posts
```
## まとめ
この記事では〇〇について説明しました。主なポイントは以下の通りです。

- ポイント1
- ポイント2
- ポイント3

次は〇〇を試してみてください。より詳しい情報は[公式ドキュメント](URL)を参照してください。
```

### Lifestyle Posts
```
## 振り返り
〇〇を始めてから、△△変化がありました。

特に効果的だったのは〇〇です。引き続き続けていきたいと思います。
```

### Learning Notes
```
## まとめ
〇〇の要点を整理しました。

- 重要概念1
- 重要概念2

理解を深めるには、実際に手を動かすのが一番です。
```

## Length Guidelines by Pattern

### Target: 800-1500 characters

**Problem-Solution:**
- Hook: 100-150 chars
- Context: 150-200 chars
- Solution: 400-600 chars
- Summary: 100-200 chars

**Introduction-Body:**
- Intro: 150-200 chars
- Body (2-3 sections): 400-600 chars each
- Summary: 100-200 chars

**Journey-Based:**
- Hook: 100-150 chars
- Phases: 300-400 chars each
- Reflection: 150-250 chars

## Choosing the Right Pattern

### Ask These Questions

1. **What's the main purpose?**
   - Teach how → Problem-Solution
   - Explain what → Introduction-Body
   - Share experience → Journey-Based

2. **What's the content type?**
   - Technical tutorial → Problem-Solution
   - Concept explanation → Introduction-Body
   - Personal story → Journey-Based

3. **What's the reader's goal?**
   - Fix something → Problem-Solution
   - Learn something → Introduction-Body
   - Relate to experience → Journey-Based
