クロード・コードにおけるGLMコーディングプランの使用方法

Claude Code は、ターミナルに常駐し、コードベースを理解し、日常的なタスクの実行、複雑なコードの説明、git ワークフローの処理など、すべて自然言語コマンドを通じて、より速くコードを作成できるエージェント コーディング ツールです。
Claude CodeはGLMコーディングプランでさらに強力になり、わずかなコストで3倍の使用量を実現します。より多くのトークンと確固たる信頼性により、より速くコーディングし、よりスマートにデバッグし、ワークフローをシームレスに管理できます。クリスマスセール： GLMコーディングプランの初回購入が50%オフ！さらに10%/20%オフ！今すぐお申し込みください。
2025年9月30日より前にサービスをご利用いただいたユーザーの皆様へ：
GLMコーディングプランのデフォルトモデルは、シームレスなユーザーエクスペリエンスを実現するGLM-4.7にアップグレードされました。
ただし、以前にGLM-4.5の固定モデルマッピングを設定していた場合はsettings.json、下記のFAQの「使用モデルの切り替え方法」セクションを参照して調整を行い、最新のGLM-4.7モデルをご使用ください。
サブスクリプションの設定が完了すると、デフォルトのサーバーモデルマッピングが適用されます。インターフェースにはClaudeモデルが表示されますが、実際にはGLMモデルが使用されます。
モデルマッピングは手動で調整できます（推奨されません）。詳細はFAQの「使用中のモデルを切り替える方法」セクションをご覧ください。

ステップ1: Claudeコードのインストール

  • 推奨されるインストール方法
  • カーソルガイドによるインストール方法
前提条件: Node.js 18以降
コピー
# Install Claude Code
npm install -g @anthropic-ai/claude-code

# Navigate to your project
cd your-awesome-project

# Complete
claude
npm に慣れていないけれど Cursor を持っているという場合は、Cursor にコマンドを入力すると、Cursor が Claude Code のインストールを案内してくれます。
コピー
https://docs.anthropic.com/en/docs/claude-code/overview Help me install Claude Code
注意: インストール中に権限の問題が発生した場合は、sudo(MacOS/Linux) を使用するか、コマンド プロンプトを管理者として実行 (Windows) して、インストール コマンドを再実行してください。

ステップ2: GLMコーディングプランの設定

1

APIキーを取得する

  • Z.AI オープン プラットフォームにアクセスし、登録またはログインします。
  • API キー管理ページで API キーを作成します。
  • 使用するために API キーをコピーします。
2

環境変数を設定する

macOS、Linux、またはWindowsで次のいずれかの方法を使用して環境変数を設定します。
注: 一部のコマンドでは、環境変数を設定するときに出力が表示されません。エラーが表示されない限り、これは正常です。
  • 自動コーディングツールヘルパー
  • 自動スクリプト
  • 手動設定
Coding Tool Helperは、 GLM Coding Planをお気に入りのCoding Toolsに素早くロードするコーディングツールコンパニオンです。インストールして起動すると、画面上のガイダンスに従ってツールのインストール、プランの設定、MCPサーバーの管理が自動的に行われます。
コピー
# Run Coding Tool Helper directly in the terminal
npx @z_ai/coding-helper
詳細については、Coding Tool Helper のドキュメントを参照してください。
ターミナルで次のコマンドを実行するだけです。
注意：macOS Linux環境のみがサポートされており、この方法はWindowsをサポートしていません。
コピー
curl -O "https://cdn.bigmodel.cn/install/claude_code_zai_env.sh" && bash ./claude_code_zai_env.sh
スクリプトは自動的に変更され~/.claude/settings.json、次の環境変数が設定されます (手動で編集する必要はありません)。
コピー
{
    "env": {
        "ANTHROPIC_AUTH_TOKEN": "your_zai_api_key",
        "ANTHROPIC_BASE_URL": "https://api.z.ai/api/anthropic",
        "API_TIMEOUT_MS": "3000000"
    }
}
Claude Code の環境変数を以前に設定している場合は、以下の手順で手動で設定できます。変更を有効にするには、新しいウィンドウを開く必要があります。
コピー
# Edit the Claude Code configuration file `~/.claude/settings.json`
# Add or modify the env fields ANTHROPIC_BASE_URL, ANTHROPIC_AUTH_TOKEN
# Note to replace `your_zai_api_key` with the API Key you obtained in the previous step

{
    "env": {
        "ANTHROPIC_AUTH_TOKEN": "your_zai_api_key",
        "ANTHROPIC_BASE_URL": "https://api.z.ai/api/anthropic",
        "API_TIMEOUT_MS": "3000000"
    }
}

ステップ3：Claude Codeから始める

設定が完了したら、ターミナルまたは cmd でClaude Code の使用を開始できます。
コピー
cd your-project-directory
claude
「この API キーを使用しますか?」というメッセージが表示されたら、「はい」を選択します。
起動後、以下に示すように、Claude Code にフォルダー内のファイルにアクセスする権限を付与します。 今すぐ開発にClaude Codeをご利用いただけます!

使用中のモデルを切り替える方法

Claude Code 内部モデル環境変数と GLM モデル間のマッピング。デフォルト構成は次のとおりです。
  • ANTHROPIC_DEFAULT_OPUS_MODEL:GLM-4.7
  • ANTHROPIC_DEFAULT_SONNET_MODEL:GLM-4.7
  • ANTHROPIC_DEFAULT_HAIKU_MODEL:GLM-4.5-Air
調整が必要な場合は、構成ファイル (たとえば、Claude Code の ~/.claude/settings.json) を直接変更して、他のモデルに切り替えることができます。
モデル マッピングをハードコーディングすると、GLM コーディング プラン モデルが更新されたときに最新のモデルに自動的に更新するのが不便になるため、通常はモデル マッピングを手動で調整することはお勧めしません。
最新のデフォルト マッピングを使用する場合は (古いモデル マッピングを構成している既存のユーザーの場合)、 のモデル マッピング構成を削除するだけでsettings.json、Claude Code は自動的に最新のデフォルト モデルを使用します。
  1. ~/.claude/settings.json次の内容で構成します。
コピー
{
  "env": {
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "glm-4.5-air",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "glm-4.7",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "glm-4.7"
  }
}
  1. 新しいターミナル ウィンドウを開いて実行し、claudeClaude Code を起動して Enter キーを押すと、/status現在のモデルの状態を確認できます。

手動設定が機能しない

構成ファイルを手動で変更した~/.claude/settings.jsonが、変更が有効にならない場合は、次のトラブルシューティング手順を参照してください。
  • すべての Claude Code ウィンドウを閉じ、新しいコマンド ライン ウィンドウを開いて、claude再度実行して開始します。
  • 問題が解決しない場合は、~/.claude/settings.jsonファイルを削除してから環境変数を再設定してください。Claude Code によって新しい設定ファイルが自動的に生成されます。
  • 構成ファイルの JSON 形式が正しいことを確認し、変数名をチェックし、欠落したコンマや余分なコンマがないことを確認します。オンライン JSON 検証ツールを使用して確認できます。

推奨されるクロードコードバージョン

Claude Codeの最新バージョンのご利用を推奨します。現在のバージョンを確認し、アップグレードするには、以下のコマンドを実行してください。
Claude Code 2.0.14 およびその他のバージョンとの互換性を確認しました。
コピー
# Check the current version
claude --version

2.0.14 (Claude Code)

# Upgrade to the latest
claude update
